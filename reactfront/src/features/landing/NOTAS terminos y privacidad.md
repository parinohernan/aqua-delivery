# ⚖️ Bases para Términos, Condiciones y Privacidad (ADM)

Este documento sirve como borrador para redactar los textos legales finales de **Aqua Delivery Manager**. Como SaaS que maneja datos de clientes y ventas de terceros, debemos ser muy claros para generar confianza.

---

## 🛠️ 1. Términos y Condiciones de Uso

### A. El Servicio (El "Software")
- Aqua Delivery Manager (ADM) provee una herramienta de gestión para la logística de repartos.
- Es un servicio **SaaS** (Software as a Service) por suscripción mensual.
- El usuario es responsable de contar con dispositivos con internet para su uso.

### B. Suscripciones y Pagos
- **Modelo de Abono:** Mensual, por adelantado.
- **Promo Lanzamiento:** Quienes se registren antes del **1 de abril** mantienen su precio promocional de por vida (mientras mantengan la suscripción activa).
- **Mora:** La falta de pago resultará en la suspensión del acceso a la plataforma tras [X] días de aviso.
- **Cancelación:** El usuario puede cancelar en cualquier momento. No hay reembolsos por períodos parciales ya pagados.

### C. Responsabilidad Limitada
- ADM no se responsabiliza por:
    - Pérdidas económicas del negocio del usuario.
    - Errores en la carga de datos por parte de los choferes.
    - Caídas del servicio por problemas de hosting externos o falta de internet del usuario.
    - Cumplimiento fiscal (IVA, facturación) del usuario (el sistema es una herramienta de gestión, no un contador).

---

## 🔒 2. Política de Privacidad y Datos

### A. Propiedad de los Datos (Punto más importante para el cliente)
- **Los datos de los clientes cargados pertenecen exclusivamente al USUARIO.**
- ADM actúa únicamente como procesador de datos.
- **Compromiso:** ADM nunca venderá ni compartirá la lista de clientes o rutas del usuario con competidores ni terceros.

### B. Datos Recolectados
- **Del Usuario:** Nombre, empresa, WhatsApp, email (para gestión de cuenta).
- **Del Negocio:** Listado de clientes, direcciones (para hoja de ruta), saldos de envases y deudas.
- **Geolocalización:** Solo se utiliza para la optimización de rutas (si el usuario activa la función).

### C. Seguridad
- Los datos se almacenan en servidores cifrados.
- Se realizan backups diarios para prevenir pérdida de información.

## 📝 3. Puntos de Conflicto a Resolver (Drafting FAQ)

*   **¿Qué pasa si el dueño quiere exportar sus datos y dejar de usar ADM?**
    - Debemos garantizar una función de "Exportación a Excel" para que el cliente no se sienta "secuestrado".
*   **¿Cómo manejamos la Ley de Protección de Datos Personales (Ley 25.326 en Arg)?**
    - El usuario (dueño de sodería) es el responsable legal de los datos de sus clientes ante la ley; ADM es el proveedor técnico.
